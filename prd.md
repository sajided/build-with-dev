Project Name: Feel-Good Calendar (Codename: VibeTime)
Status: Brainstorm / Concept

Author: Solo Founder

Tech Stack: Next.js, React, Tailwind CSS, Firebase (Firestore), Gemini API

1. Executive Summary & Philosophy
VibeTime is a conversational, brutalist-style calendar application designed to act as an active Feel-Good Productivity Coach. Instead of forcing users to manually click, drag, and type to manage a rigid calendar grid, VibeTime allows users to manage their time entirely through a chat dialogue with an embedded AI coach.

The application strictly enforces the five core pillars of Feel-Good Productivity: Prioritization, Time Blocking, Focus, Follow-Through, and Energizing Work.

Design Aesthetic

Hyper-minimalist, high-contrast, brutalist UI.

Monochromatic color palette (pitch black, stark white) with high-contrast neon-green accents for active states, focus highlights, and CTA buttons.

Data presentation inspired by printed thermal receipts.

2. User Personas & Core Workflows
Target User

The solo builder, developer, or student who suffers from choice paralysis, mindless scrolling during unstructured gaps, and execution friction (procrastination).

Primary Workflows

The Morning Dump (Planning): The user types or speaks a raw text stream of their day's variables. The AI engine processes it, negotiates priorities, and builds a realistic time-blocked schedule.

Adventure Mode (Focusing): The user enters a hyper-focused UI state to execute a specific time block.

The Interception (Unblocking): When a user triggers procrastination behavior or skips a block, the AI steps in to clear friction dynamically.

3. Functional Requirements & Feature Specifications
3.1. Conversational Auto-Blocking Engine (Prioritization & Time Blocking)

FR-1.1: The app must provide a command-line or chat-style interface for the initial daily setup.

FR-1.2: The AI must parse the raw input and enforce the selection of exactly one Daily Adventure (the primary success metric) and two Side Quests (one health-related, one relationship-related).

FR-1.3: The AI must explicitly prompt for non-work anchors (e.g., commutes, gym sessions, social plans, classes) before scheduling tasks.

FR-1.4: If the user attempts to over-schedule their day or inputs statements indicating they "don't have time," the system must trigger a specialized coaching flag: "It's not about time; it's about priority."

FR-1.5: Upon completing the conversation, the backend must instantly output a structured JSON array of time blocks and inject them directly into the Firestore database to render on the calendar grid.

3.2. "Adventure Mode" Interface (Focus)

FR-2.1: Users can activate "Adventure Mode" for any active time block.

FR-2.2: When activated, the standard calendar grid must fade out completely, isolating a single-tasking viewport containing only the current task.

FR-2.3: The interface must feature a prominent "Make it Smaller" button. Clicking this sends the current task context back to the Gemini API to break it down into atomic, micro-actions (e.g., "Build database schema" → "Write 3 lines of SQL for user table").

3.3. Procrastination SOS Chatbot (Follow-Through via the Unblock Method)

FR-3.1: If a time block is delayed, missed, or flagged manually by the user as "stuck," the app must initiate a dedicated Unblock Flow dialog.

FR-3.2: The AI coach must walk the user through the three steps sequentially (one conversational turn at a time):

Clarity: Verify if the immediate next technical action is crystal clear.

Courage: Surface hidden blockages regarding fear of failure or perfectionism.

Inertia: Trigger an automated 5-minute micro-timer (the "dose of discipline") requiring the user to simply interact with the workspace for 300 seconds.

FR-3.3: The system must prompt the user to input the name or contact shortcut of an accountability buddy for that specific task block.

3.4. Pre-Flight 3P Checklist (Energizing Work)

FR-4.1: Before starting a deep-work/difficult project block, a lightweight modal must prompt the user to quickly answer the 3 Ps:

Play: Select or define how to make the execution engaging (e.g., background audio vibe).

Power: Frame an element of the task where the user has total operational control.

People: Identify a peer or collaborator to loop in or share progress with post-session.

3.5. Thermal Receipt Analytics

FR-5.1: At the conclusion of the day, the calendar grid converts into a static, high-contrast "Day Receipt".

FR-5.2: The receipt must display:

Daily Adventure status (SUCCESS / SKIPPED).

Side Quest checkboxes.

Total focused minutes executed.

A final minimalist sign-off message generated by the AI coach.

4. Technical Architecture & Data Schema
Technology Justification

Next.js & React: Ideal for fast component-level state changes, enabling the calendar grid to update instantly as the user chats with the AI coach.

Tailwind CSS: Allows easy implementation of sharp brutalist borders, monospace typography, and custom neon utility classes.

Firebase (Firestore): Provides real-time data listeners. When the Gemini API finishes generating a schedule, writing it to Firestore updates the UI instantaneously without requiring page refreshes.

Gemini API (gemini-2.5-flash): Used for fast, low-latency text processing and structured JSON generation via responseSchema.

5. Non-Functional Requirements
Performance / Latency: The AI auto-blocking engine should parse raw user text and return the structured JSON schedule within 2.5 seconds to keep the chat interface feeling responsive.

Responsive Design: While designed primarily for desktop solo building sessions, the layout must scale down gracefully to a single-column brutalist stack on mobile devices for quick tracking on the go.

Data Security: Strict Firestore security rules to ensure users can only read/write their own daily schedules and task items.
